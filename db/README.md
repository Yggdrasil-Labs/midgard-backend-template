## 数据库目录说明

本目录存放项目相关的数据库脚本，包含表结构定义、数据变更迁移脚本以及初始化数据。

### 目录结构

*   **schema/**: 存放数据库的初始表结构定义（DDL）。
    *   用于项目初始化时的建表操作。
    *   通常包含 `create_table.sql` 等脚本。

*   **migration/**: 存放数据库的版本变更迁移脚本。
    *   用于项目迭代过程中对数据库结构的修改（如新增字段、修改索引等）。
    *   建议命名格式：`V{版本号}__{描述}.sql` (例如: `V1.0.1__add_user_column.sql`)。
    *   配合 Flyway 或 Liquibase 等工具使用。

*   **data/**: 存放数据库的初始化数据或测试数据（DML）。
    *   用于开发环境或测试环境的数据预置。
    *   包含基础配置数据、字典数据等。

### 使用建议

1.  **开发规范**: 所有数据库变更都应通过脚本文件提交，禁止直接在生产环境手动执行 SQL。
2.  **版本控制**: 迁移脚本一旦执行，不可修改。如有变更，请新增新的迁移脚本。
3.  **环境隔离**: `data/` 目录下的数据脚本应区分环境（如 `dev`, `test`, `prod`），以免误删生产数据。
