# New User Onboarding

本文件描述一个新协作者使用 Midgard 模板的最小成功路径。

## 目标

让新协作者在不破坏 COLA 分层的前提下，完成一次“从模板到业务骨架”的扩展。

## 目标用户

- 首次接触 Midgard 的后端开发者
- 新加入项目的 AI Agent
- 需要基于模板创建具体业务服务的工程师

## 核心旅程

1. 理解模块边界与命名规则。
2. 本地运行和验证模板。
3. 选择一个聚合开始扩展。
4. 在 Adapter/App/Domain/Infrastructure 中分别补齐职责。
5. 通过最小 Maven 校验确认改动有效。

## 成功标准

一个新协作者在首次扩展时，应能做到：

- 不把业务逻辑塞进 Controller
- 不让 Domain 依赖外层
- 正确使用 `InfraConvertor`
- 理解 `Cmd/Qry/CO` 的职责差异
- 使用 `./mvnw` 完成至少编译级验证

## 当前缺口

- 仓库尚无端到端的示例聚合
- 因此 onboarding 目前依赖架构说明和包级注释，而不是可运行样例
