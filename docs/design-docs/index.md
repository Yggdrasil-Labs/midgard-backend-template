# Design Docs Index

本目录存放“为什么这样设计”的长期文档，而不是一次性任务记录。

## 当前文档

| 文档 | 状态 | 用途 |
| --- | --- | --- |
| `core-beliefs.md` | active | 记录模板对智能体和人类协作者都成立的核心信念 |

## 收录规则

适合放在本目录的内容：

- 长期成立的设计原则
- 横切设计，例如多租户、鉴权、审计、事件、缓存
- 会影响多个模块和多个任务的约束

不适合放在本目录的内容：

- 单次任务的执行过程
- 临时排障笔记
- 只服务于一次 PR 的实现细节

这些内容分别进入：

- 执行过程：`docs/exec-plans/`
- 生成结果：`docs/generated/`
- 产品与用例：`docs/product-specs/`

## 维护规则

- 每篇设计文档要说明适用范围、状态、最后验证方式。
- 设计一旦失效，应及时修订或标记废弃，不要静默腐烂。
- 若实现改变了长期设计，PR 应同时更新这里的文档。

当前最小元数据建议：

- `状态`
- `适用范围`
- `最后验证`
