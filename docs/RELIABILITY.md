# Reliability

本文件记录模板级可靠性约束，关注的是“未来基于该模板构建的服务应该默认具备什么可靠性意识”。

## 当前仓库事实

- 已有 Spring Boot 多环境配置文件
- 具备分层架构与 Maven 构建骨架
- 尚未提交完整的可观测性、压测或 SLO 配置

## 模板级原则

### 1. 可失败，但要可定位

对于后续服务实现，应优先保证：

- 关键失败可以从日志、异常边界和上下文中定位
- 技术异常不直接泄漏到协议层
- 应用启动和关键链路可被验证

### 2. 事务边界要集中

- 事务优先放在 App 层
- 避免 Controller、Domain、Repository 各自隐式控制流程

### 3. 外部依赖要隔离

- 第三方能力走 Gateway
- 持久化走 Repository
- 不让核心规则直接耦合外部系统语义

### 4. 模板中的可靠性文档要诚实

没有落地的能力不要写成“已支持”。

当前不能宣称已有：

- 完整 tracing
- 自动化 SLA 检查
- 标准化恢复演练

## 后续建议

- 增加统一日志字段约定
- 明确健康检查和 readiness 要求
- 当引入 MQ 或外部调用样例时，补充超时、重试、幂等等约束
