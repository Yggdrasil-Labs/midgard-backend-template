# Tech Debt Tracker

本文件记录当前仓库已知但尚未处理的结构性欠账。

## 使用方式

- 每条债务都要有明确范围和下一步，而不是模糊抱怨。
- 如果某项债务已经进入执行计划，应链接到对应计划文件。
- 如果债务已解决，要删除或标记关闭日期，避免堆积陈尸记录。
