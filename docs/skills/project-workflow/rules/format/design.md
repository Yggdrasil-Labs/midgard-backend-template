# Design 文档规格

design.md 只回答：**怎么做**。承接 spec（做什么），输出给 plan（怎么拆）。

## 边界判断

| spec | design | plan |
|------|--------|------|
| 做什么、为什么 | 怎么做 | 怎么拆、谁执行 |
| 用户场景、验收标准 | 数据模型、接口契约、核心流程 | 任务列表、依赖、验证命令 |
| 产品约束 | 技术约束、性能约束 | 风险、阻塞、完成标准 |
| 功能边界 | 影响范围 | 执行模式 |

## 必填章节

1. **背景**（一段话概括动机）
2. **技术方案**（三子节缺一不可）：
   - 数据模型：实体、字段、关系
   - 接口契约：API / 函数签名 / 消息格式
   - 核心流程：Mermaid 主流程图
3. **影响范围**（模块/文件 + 变更类型）
4. **约束**（继承 spec + 叠加技术约束，每条可验证）
5. **迁移与兼容**（不涉及写"不适用"）
6. **发布与回滚**
7. **观测性**
8. **异常处理**
9. **验证方式**（单元 + 集成 + 性能）
10. **备选方案**（至少一个）
