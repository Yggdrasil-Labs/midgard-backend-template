# 需求对齐（Grill）步骤约束

在分级和创建文档之前，逐一追问决策树的每个分支，直到达成共识。

## 规则

1. **逐个提问**：每次只问一个问题，等待用户回答后再进入下一个
2. **给出推荐答案**：每个问题附带基于代码库推断的推荐选项
3. **代码能回答的先查代码**：可以通过阅读现有代码/配置得到答案的，先读代码
4. **覆盖关键分支**：
   - 功能边界（做什么 / 不做什么）
   - 用户场景的分支路径
   - 技术约束与兼容性
   - 数据模型变更范围
   - 依赖关系和影响面
5. **终止条件**：所有关键分支都有明确答案，无模糊地带

## 反模式

- 一次问多个问题
- 不附带推荐就提问
- 问开放式问题（"你想怎么做？"）
- 不读代码就问用户已有答案的问题
