# 上下游映射约束

文档之间的映射关系必须完整，遗漏视为 P0 问题。

## spec → design

| spec 中的 | 必须映射到 design 中的 |
|-----------|----------------------|
| 每个用户场景 | 核心流程（Mermaid） |
| 每个输入/输出 | 接口契约 |
| 每个异常场景 | 异常处理表 |
| 每条产品约束 | 约束（继承 + 叠加技术约束） |
| 每条验收标准 | 验证方式 |

## design → plan

| design 中的 | 必须映射到 plan 中的 |
|-------------|---------------------|
| 影响范围表每一行 | 至少一个 Task |
| 模块间依赖关系 | Task 的 depends_on |
| 验证方式 | Step 中的 Run 命令 |
| 迁移与兼容 | 前置 Task |
| 接口契约 | 测试用例中的 expected |

## 检查方法

逐行对照上游文档，确认每个要素在下游都有着陆点。遗漏项记为 P0。
