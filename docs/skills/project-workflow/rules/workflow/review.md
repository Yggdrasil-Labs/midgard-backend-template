# 审查流程约束

每份文档生成后必须经过审查。评分标准见 `format/review.md`。

## 流程

1. 文档生成
2. 审查评分（可使用 subagent，审查者与作者分离）
3. 达标（均分 ≥ 4.0 且无 P0/P1）→ PASS
4. 未达标 → 修复 → 重审（最多 2 次修复）
5. PASS 后向用户呈现审查摘要（总分 + 关键发现），用户确认后进入下游

## 审查者行为准则

- 基于事实评分，引用原文具体章节
- 每个 P0/P1 必须有修复建议
- 默认立场是"这份文档有问题"——找问题，不是确认正确
- 评分诚实，宁可打回重改不放水

## 审查轨迹

通过后在文档末尾附加：

```markdown
<!-- review-trace
score: {均分}/5.0 ✅
findings: P0={n} P1={n} P2={n} P3={n}
iterations: {N}
-->
```
