# Security

本文件描述模板级安全基线，而不是宣称仓库已经实现了完整安全体系。

## 当前仓库事实

- 这是后端模板仓库
- 当前未看到完整的鉴权、审计、密钥管理或安全测试脚本
- 因此这里记录的是基线要求与未来落点

## 基线要求

### 1. 分层边界也是安全边界

- Adapter 负责输入校验与协议约束
- Domain 不信任外部输入已经“天然正确”
- Infrastructure 负责凭据、外部系统和数据库访问的技术隔离

### 2. 不在仓库中硬编码敏感信息

- 账号、密钥、令牌、连接串不得硬编码提交
- 配置示例中使用占位值，不使用真实密钥

### 3. 输出对象最小暴露

- CO 只暴露外部需要的字段
- 不把内部标识、敏感字段或技术字段直接透出

### 4. 参数校验不是业务校验的替代

- `@Valid` 解决格式问题
- 领域规则仍需在 Domain 中表达

## 当前缺口

- 缺少明确的鉴权/鉴权扩展设计文档
- 缺少安全相关测试样板
- 缺少配置项敏感信息管理约定说明

这些缺口不应被忽视，也不应被文档粉饰。
