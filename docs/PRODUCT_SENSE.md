# Product Sense

Midgard 不是终端产品，而是后端模板。

因此这里的 product sense 不围绕“增长”或“页面转化”，而围绕模板是否真的帮助协作者更快、更稳地构建服务。

## 模板的核心价值

- 缩短从零到可演化服务的时间
- 降低破坏分层边界的概率
- 让新协作者和 Agent 快速形成统一心智模型

## 做决策时优先考虑

1. 是否提升了模板的可迁移性。
2. 是否让新协作者更容易做出正确的分层决策。
3. 是否降低了后续扩展的熵。

## 不优先的方向

- 只为了展示效果而增加复杂样板
- 在模板中提前塞入过多业务假设
- 引入与当前技术栈不一致的重量级方案

## 评估一个新能力是否值得进入模板

- 是否会被大多数服务复用
- 是否能形成稳定约束而不是一次性技巧
- 是否会增加理解成本
- 是否能被文档和验证手段清楚表达
